## Feature Engineering

Feature Engineering is the process of creating, modifying, or selecting features from raw data to improve machine learning model performance. It helps models learn meaningful patterns from the data and often increases prediction accuracy.

## Feature Creation

Feature Creation involves generating new features from existing data using mathematical operations or domain knowledge. These new features provide additional information that can improve model performance.

## Feature Transformation

Feature Transformation is the process of changing the format or representation of existing features. It helps make the data more suitable for machine learning algorithms by improving its distribution or scale.

## Feature Scaling

Feature Scaling is the process of bringing numerical features to a similar range or scale. It improves the performance of algorithms such as Logistic Regression, KNN, and SVM by preventing features with larger values from dominating the model.

## Feature Encoding

Feature Encoding is the process of converting categorical data into numerical values so that machine learning algorithms can process it. Common encoding techniques include Label Encoding and One-Hot Encoding.

## Feature Binning

Feature Binning is the process of grouping continuous numerical values into a limited number of intervals or categories. It simplifies the data, reduces noise, and can help improve the performance and interpretability of some machine learning models.

### Key Learning
- Importance of feature engineering in Machine Learning.
- Creating new features from existing data.
- Transforming features to improve data distribution.
- Scaling data using StandardScaler and MinMaxScaler.
- Encoding categorical variables into numerical values.
- Using feature binning to categorize continuous data.

### Summary

Today, I learned about *Imbalanced Data*, a common problem in machine learning where one class has significantly more samples than the other. This can cause models to become biased toward the majority class and produce misleading accuracy. We visualized the class distribution and learned different techniques to balance the dataset.

### **SMOTE (Synthetic Minority Over-sampling Technique)**

SMOTE (Synthetic Minority Over-sampling Technique) is a data balancing method used to handle imbalanced datasets. It creates new synthetic samples for the minority class instead of simply duplicating existing ones. This helps improve the performance of machine learning models by reducing bias toward the majority class and enabling better prediction of minority class instances.

### Key Learning
- Understanding the problem of imbalanced datasets.
- Learning oversampling and undersampling techniques.
- Using SMOTE and ADASYN to generate synthetic minority samples.
- Applying Tomek Links to remove overlapping data points.
- Improving machine learning model performance with balanced datasets.
